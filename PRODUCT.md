# Product

## Register

product

## Users

Field sales associates at Syook using the app on their iPhone during and between client meetings. One-handed use is common. High task frequency, short sessions — open the app, log a stage change, check what needs attention, close. Low tolerance for friction.

## Product Purpose

A lightweight mobile CRM for individual reps to track their active deals, log stage changes with supporting documents, flag stalled deals, and manage on-hold accounts. Success means the rep can complete any core action in under three taps from the home screen.

## Brand Personality

Precise, calm, trustworthy. The app should feel like a well-made native iOS tool — familiar enough that no training is needed, polished enough that using it feels good.

## Anti-references

- Salesforce Mobile: dense, heavy, overwhelming for a field rep
- Generic React web app styled for mobile: bordered cards, web typography, non-native patterns
- Over-animated dashboards with hero metrics and gradient accents

## Design Principles

1. Disappear into the task. The interface should never draw attention to itself. No decoration for its own sake.
2. iOS-native vocabulary. Use patterns the OS already taught the user: grouped table views, bottom sheets, navigation bars, action buttons. No invented affordances.
3. Touch confidence. Every tappable target meets 44pt minimum. Tap feedback is immediate and physical.
4. Hierarchy through restraint. One accent color, used only for primary actions and active state. All other color is semantic (warning, success, neutral).
5. Motion earns its place. Transitions convey state change. Nothing animates just to look alive.

## Accessibility & Inclusion

WCAG AA minimum. All body text at 4.5:1+ contrast. Touch targets 44pt+. Support for reduced motion via prefers-reduced-motion. No color-only state communication.
